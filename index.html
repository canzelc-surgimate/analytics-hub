Technical Architecture

Technology Stack & Infrastructure

Real-time event-driven pipeline where Rails, FHIR, and Connect publish resource changes to a shared SNS topic, fanning out to dedicated SQS queues and Lambda functions (Python/Go) for OpenSearch reindexing, ClickHouse analytics, and FHIR notifications. Lambdas read full records from a PostgreSQL Reader Instance, with results visualized via Apache Superset and embedded into the Surgimate Angular application.

Data Strategy

Historical Backfill & Real-Time Streaming

Historical Backfill
  • One-time full ingestion of all historical data
  • Kick off per-client via Rails rake job, isolated to ClickHouse
  • (Nice-to-have) UI in Surgimate Admin to trigger client extractions
Real-Time Streaming
  • Rails publishes events to SNS on every create/update
  • SQS buffers with retry and dead-letter queue
  • Lambda transforms and writes to ClickHouse in real time
  • Intelligent schema for efficient joins

Execution Risks

Mitigation Strategy
  • Parallel development across 3 engineers

  • Focus on functionality over perfection
  • Weekend 1 front-loads all infrastructure risk
